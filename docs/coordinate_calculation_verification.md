<ymin>433</ymin>

```

## 缩放到512×512

```
scale_x = 512 / 800 = 0.64
scale_y = 512 / 800 = 0.64
```

### 预期的像素坐标 (Pascal VOC)
```
xmin_scaled = 498 × 0.64 = 318.72 ≈ 319
ymin_scaled = 433 × 0.64 = 277.12 ≈ 277

ymax_scaled = 574 × 0.64 = 367.36 ≈ 367

宽度: 376 - 319 = 57
高度: 367 - 277 = 90
中心X: 319 + 57/2 = 347.5
中心Y: 277 + 90/2 = 322

## 对比分析

| 维度 | 预期值 | 实际值 | 偏差 |
|------|--------|--------|------|
| x_center | 347.5 | 505.0 | **+157.5px** |
| y_center | 322.0 | 472.5 | **+150.5px** |
| width | 57 | 14 | **-43px** |
| height | 90 | 79 | **-11px** |

### 🚨 发现的问题

1. **中心点严重偏移**: 
   - X方向偏移 +157.5px (相当于512图像的31%!)
   - Y方向偏移 +150.5px (相当于512图像的29%!)

2. **宽度严重缩小**:
   - 预期宽度 57px，实际只有 14px (缩小了75%!)

3. **实际坐标接近图像边界**:
   - xmax = 512 (刚好在右边界)
   - ymax = 512 (刚好在下边界)

---

## 🔍 问题根源分析

### 猜测: YOLO标注没有缩放，仍然是原始800×800的坐标

让我验证这个假设：

**如果YOLO标签仍是基于800×800**:
```
x_center_px_800 = 0.986328125 × 800 = 789.0625
y_center_px_800 = 0.9228515625 × 800 = 738.28125
width_px_800 = 0.02734375 × 800 = 21.875
height_px_800 = 0.154296875 × 800 = 123.4375

# 这些值也不对，不是原始的542.5, 503.5, 89, 141
```

### 猜测2: 坐标转换时使用了错误的图像尺寸

可能在 `common_to_yolo()` 函数中，使用了错误的图像尺寸参数。
