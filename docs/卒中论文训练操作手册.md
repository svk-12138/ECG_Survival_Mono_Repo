# 卒中论文训练操作手册

这份手册是给医生或学生直接使用的训练说明，默认按 Win11 场景来写。  
目标很简单：你只需要改一个配置文件，然后运行一个启动脚本，就可以开始训练。

如果你手里的数据还没有整理成训练可读的 `manifest.json`，请先看：

- [docs/卒中论文数据处理操作手册.md](卒中论文数据处理操作手册.md)

## 先记住最短操作流程

医生真正需要做的事情只有下面 3 步：

1. 打开 `configs\train_stroke_thesis.env`（没有就先复制 `.example` 文件）
2. 修改里面的几个参数
3. 双击运行 `scripts\train_stroke_thesis.bat`

**不要改 `.bat` 和 `.ps1` 脚本本身**，它们是固定的启动器，不需要动。

## 第 1 步：准备配置文件

配置文件路径：`configs\train_stroke_thesis.env`

如果这个文件不存在，先复制模板：

```bat
copy configs\train_stroke_thesis.env.example configs\train_stroke_thesis.env
```

或者在文件管理器里手动复制 `train_stroke_thesis.env.example`，改名为 `train_stroke_thesis.env`。

> 注意：`train_stroke_thesis.env` 已被 `.gitignore` 忽略，不会被提交到 GitHub，
> 所以你在本地改的路径不会影响其他人。

## 第 2 步：修改配置文件

用记事本或任意文本编辑器打开 `configs\train_stroke_thesis.env`，只需要关注下面几个参数。

### 必改参数（3 个）

#### 1. 数据清单路径

```env
MANIFEST="D:\your\path\stroke_manifest.json"
```

改成你本地的 `manifest.json` 路径，例如：

```env
MANIFEST="D:\stroke_data\processed\manifest.json"
```

#### 2. ECG 数据目录（XML 和 CSV 二选一）

```env
XML_DIR="D:\your\path\xml_dir"
CSV_DIR=
```

- 原始心电是 XML 格式，就填 `XML_DIR`，把 `CSV_DIR` 留空
- 已经整理成 CSV，就填 `CSV_DIR`，把 `XML_DIR` 留空
- 两个都填会报错，两个都不填也会报错

#### 3. 导联模式

```env
LEAD_MODE=8lead
```

现在支持：

- `8lead`：I, II, V1-V6（你的数据只有 8 导，**请用这个**）
- `12lead`：I, II, III, aVR, aVL, aVF, V1-V6

### 常用参数（按需修改）

#### 任务类型

```env
TASK_MODE=prediction
```

- `prediction`：生存预测，论文主实验
- `classification`：二分类，对照实验

#### 模型选择（新）

现在推荐使用 `MODEL_NAME` 一键选择预设模型，不需要再手动调整 batch、dropout 等参数：

```env
MODEL_NAME=resnet_small
```

可选预设：

| MODEL_NAME | 参数量 | 适用场景 | 推荐 |
|------------|--------|---------|------|
| `tcn_light` | ~25,000 | 1200 样本 | 本地测试 |
| `resnet_small` | ~12 万 | 1 万样本 | **医生端推荐** |
| `resnet_standard` | ~294 万 | 10 万+样本 | 论文同款 |
| `cnn_transformer` | ~69 万 | 实验性 | 不推荐 |

> 设置 `MODEL_NAME` 后，`MODEL_TYPE` 会被自动忽略。

#### 固定数据集划分（新）

```env
SPLIT_FILE="outputs/stroke_survival_thesis/dataset_split.json"
```

- **首次训练**：自动随机划分 train/val/test，并保存到该文件
- **后续训练**：自动加载该文件，确保每次使用完全相同的数据集组成
- **想重新划分**：删除该文件后重新训练即可
- **留空**：每次随机划分（不推荐，不同实验结果不可比）

#### 预测时间窗口

```env
MAX_TIME=1825.0
PREDICTION_HORIZON=1825.0
```

单位是天。根据你的数据随访时长选择：

| 随访时长 | 推荐设置 | 说明 |
|---------|---------|------|
| 本地 1200 样本（平均 3.7 年） | `MAX_TIME=1095.0` | 3 年窗口，样本最充足 |
| 医院端 1 万样本（随访 12 年） | `MAX_TIME=1825.0` | 5 年窗口，覆盖 79% 事件 |

#### 交叉验证

```env
CV_FOLDS=1
```

- `1`：固定留出法（按 TRAIN/VAL/TEST_RATIO 划分）
- `5`：5 折交叉验证（小样本推荐，结果更稳定，但不支持固定划分文件）

### 完整配置示例

**医院端 1 万样本推荐配置：**

```env
MANIFEST="D:\stroke_data\processed\manifest.json"
XML_DIR="D:\stroke_data\xml"
CSV_DIR=
TASK_MODE=prediction
MODEL_NAME=resnet_small
LEAD_MODE=8lead
MAX_TIME=1825.0
PREDICTION_HORIZON=1825.0
SPLIT_FILE="outputs/stroke_survival_thesis/dataset_split.json"
TRAIN_RATIO=0.7
VAL_RATIO=0.15
TEST_RATIO=0.15
CV_FOLDS=1
```

**本地 1200 样本推荐配置：**

```env
MANIFEST="D:\stroke_data\processed\manifest.json"
XML_DIR="D:\stroke_data\xml"
CSV_DIR=
TASK_MODE=prediction
MODEL_NAME=tcn_light
LEAD_MODE=8lead
MAX_TIME=1095.0
PREDICTION_HORIZON=1095.0
SPLIT_FILE="outputs/stroke_survival_thesis/dataset_split.json"
CV_FOLDS=5
```

## 第 3 步：启动训练

在项目根目录双击，或在命令行里运行：

```bat
scripts\train_stroke_thesis.bat
```

如果你习惯 PowerShell，也可以运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\train_stroke_thesis.ps1
```

Linux / macOS / WSL 用户运行：

```bash
bash scripts/train_stroke_thesis.sh
```

## 第 4 步：查看训练结果

默认输出目录：

```text
outputs\stroke_survival_thesis
```

训练完成后先看这几个文件：

| 文件 | 说明 |
|------|------|
| `training_metrics.csv` | 每轮训练的 loss / AUROC / C-index |
| `run_summary.json` | 本次训练汇总（最优指标、权重路径） |
| `model_best.pt` | 验证集最优权重 |
| `predictions/val_risk_scores.csv` | 验证集每个样本的风险分数 |
| `dataset_split.json` | 固定划分文件（保存 train/val/test 的患者 ID 列表） |

写论文时至少保留：模型权重、训练指标表、关键参数记录、`dataset_split.json`。

## 常见实验组合

| 实验 | TASK_MODE | MODEL_NAME | LEAD_MODE | 用途 |
|------|-----------|------------|-----------|------|
| 医院端主实验 | prediction | resnet_small | 8lead | 1 万样本主结果 |
| 本地验证 | prediction | tcn_light | 8lead | 1200 样本快速验证 |
| 对照实验 | classification | resnet_small | 8lead | 与主实验对比 |
| 大样本论文 | prediction | resnet_standard | 8lead | 10 万+样本 |

## 常见问题

**提示 `Manifest 不能为空`**  
→ `MANIFEST` 路径没填，或文件不存在，检查路径是否正确。

**提示 `XmlDir 和 CsvDir 只能填一个`**  
→ `XML_DIR` 和 `CSV_DIR` 同时填了，把不用的那个留空。

**提示 `未知的模型预设`**  
→ 检查 `MODEL_NAME` 的拼写，可选值：`tcn_light`、`resnet_small`、`resnet_standard`、`cnn_transformer`。

**提示导联不够或读不进去**  
→ 你的数据只有 8 导，把 `LEAD_MODE` 改成 `8lead`。

**训练很慢**  
→ 检查是否有 GPU，如果有，确认 `DEVICE` 没有被设置成 `cpu`。  
→ 也可以先把 `EPOCHS` 改小（比如 10）跑一个快速验证。

**想切换模型重新跑，但保持相同的数据集划分**  
→ 只改 `MODEL_NAME` 这一行，保留 `SPLIT_FILE` 路径不变，重新运行 `.bat` 即可。  
→ 程序会自动加载之前保存的划分文件，确保对比实验使用完全相同的数据。

**想重新随机划分数据集**  
→ 删除 `SPLIT_FILE` 指向的 `dataset_split.json` 文件，重新运行 `.bat`。

**查看所有可用模型预设**  
→ 在命令行运行：`python scripts/run_survival_training.py --list-models`
