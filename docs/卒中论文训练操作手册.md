# 卒中论文训练操作手册

这份手册是给医生或学生直接使用的训练说明，默认按 Win11 场景来写。  
目标很简单：你只需要改一个参数文件，然后运行一个启动脚本，就可以开始训练。

如果你手里的数据还没有整理成训练可读的 `manifest.json`，请先看：

- [docs/卒中论文数据处理操作手册.md](卒中论文数据处理操作手册.md)

数据处理完成后，训练脚本会自动读取 `processed\training_inputs.json`，这样很多路径就不用再手工填写。

## 先记住最短操作流程

医生真正需要做的事情只有下面 3 步：

1. 打开 [scripts/train_stroke_thesis.ps1](../scripts/train_stroke_thesis.ps1)
2. 修改脚本最上面的几个参数
3. 双击或运行 `scripts\train_stroke_thesis.bat`

请不要修改 `train_stroke_thesis.bat`，它只是负责帮你启动 PowerShell 脚本。

## 训练前你手里应该有什么

建议至少准备好下面这些内容：

- 一个训练清单文件 `manifest.json`
- 一份 ECG 数据目录：XML 或 CSV 二选一
- 每条数据对应的结局时间 `time`
- 每条数据对应的结局事件 `event`，通常是 `0/1`

如果你已经先做过数据处理，一般会得到一个 `processed` 文件夹，里面常见的关键文件有：

- `processed\manifest.json`
- `processed\training_inputs.json`
- `processed\处理报告.txt`

这种情况下，训练会更省事，因为脚本会自动把 `manifest`、数据目录、导联建议这些信息读进来。

## 第 1 步：打开真正要改的文件

请打开：

- [scripts/train_stroke_thesis.ps1](../scripts/train_stroke_thesis.ps1)

不要改下面这个文件：

- [scripts/train_stroke_thesis.bat](../scripts/train_stroke_thesis.bat)

原因很简单：

- `.ps1` 是参数文件，也是训练启动脚本
- `.bat` 只是为了让 Win11 下启动更方便

## 第 2 步：优先看脚本顶部这些参数

### 必须确认的 4 类参数

#### 1. 数据清单路径

```powershell
$Manifest = "D:\your\path\stroke_manifest.json"
```

这表示训练要读取哪一个 `manifest.json`。

如果你已经运行过数据处理脚本，并且仓库附近存在 `processed\training_inputs.json`，这里通常会被自动替换为正确路径。  
也就是说，很多时候你不需要再手工改它，只需要检查一下是否正确即可。

#### 2. ECG 数据目录

XML 和 CSV 二选一，只能填一个：

```powershell
$XmlDir = "D:\your\path\xml_dir"
$CsvDir = ""
```

或者：

```powershell
$XmlDir = ""
$CsvDir = "D:\your\path\csv_dir"
```

简单理解：

- 原始心电是 XML，就填写 `$XmlDir`
- 已经整理成 CSV，就填写 `$CsvDir`
- 两个都填会报错
- 两个都不填也会报错

#### 3. 任务类型

```powershell
$TaskMode = "prediction"
```

这里最重要，因为现在项目支持两种任务：

- `prediction`：生存预测，适合作为论文主实验
- `classification`：二分类，适合作为对照实验

如果你的毕业论文重点是“预测卒中风险/结局风险”，建议主实验先用 `prediction`。

#### 4. 导联模式

```powershell
$LeadMode = "12lead"
```

现在支持：

- `8lead`
- `12lead`

简单理解：

- 如果你的数据稳定支持 12 导，可以优先用 `12lead`
- 如果部分数据没有完整 12 导，或者数据处理报告建议使用 8 导，就改成 `8lead`

### 建议一起确认的训练划分参数

```powershell
$TrainRatio = 0.8
$ValRatio = 0.2
$TestRatio = 0.0
$CVFolds = 1
```

这组参数的含义很重要：

- `TrainRatio`：训练集比例
- `ValRatio`：验证集比例
- `TestRatio`：测试集比例
- `CVFolds`：是否启用交叉验证

当前默认设置表示：

- 80% 训练
- 20% 验证
- 不单独留测试集
- 使用固定留出法，不做 K 折交叉验证

对于只有 1200 条卒中数据的小样本场景，这个默认值是比较稳妥的起点。

请特别注意：

- 当 `$CVFolds = 1` 时，使用上面的训练/验证/测试比例
- 当 `$CVFolds > 1` 时，会切换成交叉验证
- 一旦切换成交叉验证，`TrainRatio / ValRatio / TestRatio` 会被忽略

## 第 3 步：推荐你直接照着改

### 情况 1：你已经先做过数据处理

如果你已经有 `processed\training_inputs.json`，最省事的写法通常是：

```powershell
$Manifest = "D:\your\path\stroke_manifest.json"
$XmlDir = "D:\your\path\xml_dir"
$CsvDir = ""
$TaskMode = "prediction"
$LeadMode = "12lead"
$TrainRatio = 0.8
$ValRatio = 0.2
$TestRatio = 0.0
$CVFolds = 1
```

实际运行时，脚本会优先尝试自动读取：

- `manifest`
- `xml_dir / csv_dir`
- `recommended_lead_mode`
- `waveform_type`

所以你真正最需要盯住的是：

- 任务类型是不是 `prediction` 还是 `classification`
- 导联模式是不是 `8lead` 还是 `12lead`
- 数据划分比例是不是符合你的实验设计

### 情况 2：你还没有做过数据处理

那你就必须手工把路径写完整，例如：

```powershell
$Manifest = "D:\stroke_project\processed\manifest.json"
$XmlDir = "D:\stroke_project\xml_dir"
$CsvDir = ""
$TaskMode = "prediction"
$LeadMode = "12lead"
```

路径可以这样写：

- 绝对路径：`D:\stroke_project\processed\manifest.json`
- 相对路径：`data\stroke_manifest.json`

如果路径里有空格或中文，不用慌，只要保留外面的双引号即可。

## 第 4 步：启动训练

在项目根目录双击，或在命令行里运行：

```bat
scripts\train_stroke_thesis.bat
```

如果你习惯 PowerShell，也可以运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\train_stroke_thesis.ps1
```

对 Win11 医生用户来说，最推荐的仍然是 `.bat` 方式，因为更简单，中文输出也已经做了编码处理。

## 跑完后结果去哪里看

默认输出目录是：

```text
outputs\stroke_survival_thesis
```

一般先看下面几个文件：

- `model_final.pt`：最终模型权重
- `training_metrics.csv`：训练过程指标
- `best_threshold.json`：分类阈值结果
- `cv_summary.json`：交叉验证汇总结果，仅在 K 折时出现

如果你是写毕业论文，通常至少要保留：

- 模型输出目录
- 训练指标表
- 关键参数截图或参数记录

## 医生最常用的 4 种实验设置

### 1. 论文主实验：生存预测 + 12 导

```powershell
$TaskMode = "prediction"
$LeadMode = "12lead"
```

### 2. 导联对照实验：生存预测 + 8 导

```powershell
$TaskMode = "prediction"
$LeadMode = "8lead"
```

### 3. 分类对照实验：二分类 + 12 导

```powershell
$TaskMode = "classification"
$LeadMode = "12lead"
```

### 4. 简化基线实验：二分类 + 8 导

```powershell
$TaskMode = "classification"
$LeadMode = "8lead"
```

## 对 1200 条卒中数据的推荐起始参数

先建议从下面这组开始：

```powershell
$TaskMode = "prediction"
$LeadMode = "12lead"
$NIntervals = 20
$TrainRatio = 0.8
$ValRatio = 0.2
$TestRatio = 0.0
$CVFolds = 1
$PredictionHorizon = "null"
$ApplyFilters = $true
```

这组参数的意思是：

- 用生存预测作为主任务
- 优先跑 12 导模型
- 按 80% 训练、20% 验证来划分
- 暂时不额外切出测试集，避免样本太少
- 打开 ECG 滤波，补上基础杂波处理

## 训练过程中最容易遇到的 5 个问题

### 1. 提示 `Manifest 不能为空`

说明训练清单路径没有填好，或者自动读取失败。  
先检查：

- `processed\training_inputs.json` 是否存在
- `$Manifest` 是否写成了真实路径

### 2. 提示 `XmlDir 和 CsvDir 只能填一个`

说明 XML 和 CSV 目录同时填写了。  
保留一个，把另一个改成空字符串 `""`。

### 3. 提示 `XmlDir 和 CsvDir 至少要填一个`

说明两个都没填。  
请根据你的 ECG 数据格式，至少填写一个目录。

### 4. 提示导联不够或模型读不进去

这通常说明：

- 数据并不是每条都支持 12 导
- 或者当前波形类型下导联不完整

先回去查看数据处理输出里的：

- `处理报告.txt`
- `lead_audit.csv`

必要时把 `$LeadMode` 改成 `8lead`。

### 5. 比例设置后感觉没生效

先检查 `$CVFolds`。  
如果它大于 1，就说明你已经切换成交叉验证了，这时比例参数不会再生效。
