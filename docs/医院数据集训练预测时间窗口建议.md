## 三、推荐方案

### 方案A：3年训练 + 3年预测（推荐）

**配置参数**:
```yaml
max_time: 1095.0              # 3年 = 1095天
prediction_horizon: 1095.0    # 预测3年风险
n_intervals: 15               # 15个区间（每2.4个月一个）
```

**理由**:
1. **事件充足**: 前3年累计561例事件（58.4%），样本量充足
2. **临床意义**: 3年卒中风险是临床常用预测时间点
3. **数据质量**: 避免长期随访带来的失访偏倚
4. **模型稳定**: 事件密度高（平均187例/年），模型易学习

**适用场景**:
- 初步验证模型可行性
- 快速迭代实验
- 与传统ML基线对比（你的XGBoost在3年窗口AUC=0.72）

---

### 方案B：5年训练 + 5年预测（平衡方案）

**配置参数**:
```yaml
max_time: 1826.25             # 5年 = 1826.25天
prediction_horizon: 1826.25   # 预测5年风险
n_intervals: 20               # 20个区间（每3个月一个）
```

**理由**:
1. **事件覆盖**: 前5年累计761例事件（79.2%），覆盖绝大多数
2. **国际标准**: 5年风险是心血管领域最常用的预测窗口
3. **论文对标**: 葛均波EHJ论文使用5年预测窗口
4. **统计功效**: 事件数足够支撑深度学习模型训练

**适用场景**:
- 正式论文发表
- 与国际研究对比
- 临床应用部署

---

### 方案C：7年训练 + 5年预测（保守方案）

**配置参数**:
```yaml
max_time: 2555.0              # 7年 = 2555天
prediction_horizon: 1826.25   # 预测5年风险
n_intervals: 25               # 25个区间
```

**理由**:
1. **最大化事件**: 前7年累计852例事件（88.7%）
2. **长期随访**: 利用更长随访数据
3. **分层预测**: 可以同时评估3年、5年、7年风险

**缺点**:
- 第6-7年事件稀疏（45+46=91例），模型难以学习后期模式
- 长期随访失访率高，数据质量下降
- 训练时间更长

---

## 四、不推荐的方案

### ❌ 方案D：10年训练 + 10年预测

**问题**:
1. 第8-12年仅54例事件（5.6%），极度稀疏
2. 第12年event=0，说明数据截断或随访结束
3. 模型会过度拟合前期数据，无法学习后期模式
4. 临床上10年预测意义有限（患者情况变化大）

## 五、最终建议

### 给医生的建议

**阶段1：快速验证（当前）**
- 使用**方案A（3年）**
- 目标：验证深度学习可行性，对比XGBoost基线

```bash
# 或直接Python
python scripts/run_survival_training.py --config configs/stroke_survival_thesis.yaml
```

### 6.3 评估指标

模型会自动输出：
- **C-index**: 生存分析的标准指标（越高越好，>0.7为良好）
- **AUC**: 二分类性能（越高越好，>0.75为良好）
- **Brier Score**: 校准度（越低越好，<0.15为良好）

## 八、常见问题

**Q1: 为什么不用全部12年数据？**  
A: 第8-12年仅54例事件（5.6%），数据极度稀疏，模型无法学习有效模式，反而会降低前期预测准确性。

**Q2: 3年和5年预测哪个更好？**  
A: 3年适合快速验证，5年适合正式发表。建议先用3年跑通流程，再用5年做最终论文。

**Q3: 如何处理随访不足的样本？**  
A: 代码已自动处理：
- event=1且time<=窗口：正样本
- event=0且time>=窗口：负样本
- event=0且time<窗口：排除（随访不够长）

**Q4: 本地1200样本能用5年窗口吗？**  
A: 不建议。本地数据平均随访3.7年，用5年窗口会排除大量样本，导致样本量不足。

**Q5: 医生端11606样本应该用什么模型？**  
A: 
- 1万样本：ResNet1d小版（12万参数）或多任务学习
- 参考 `docs/研究路线图_1200样本到400万样本.md` 第2阶段
